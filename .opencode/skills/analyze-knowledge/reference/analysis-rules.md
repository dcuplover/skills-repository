# 知识分析规则

## 什么算"新知识"

- 知识库中不存在任何与之相关的文档
- 现有文档中没有涵盖该知识点
- 属于全新的主题领域

## 什么算"冲突"

- 与已有知识在结论上矛盾（例如：旧文档说"推荐使用方式A"，新内容说"方式A有严重问题，应使用方式B"）
- 同一主题但给出不同的最佳实践
- 数据或事实不一致
- 版本更新导致旧知识可能过时

## 什么算"已知且一致"

- 知识库中已有相同或高度类似的内容
- 新内容只是换了一种说法，但结论一致
- 可以在现有文档 sources 中追加来源

## 判断优先级

1. 先判断是否冲突（最重要，宁多勿少）
2. 再判断是否已知
3. 剩余的视为新知识

## 粒度建议

- 一个知识文档应聚焦于一个明确的主题
- 如果一篇文章包含多个不同主题的知识点，拆分为多个知识文档
- topic 命名使用英文短横线：`python-decorator-patterns`、`react-hooks-best-practices`
